What it does

Built for devs who live with Docker and AI coding agents.

Two cleaners no other macOS tool does well — volume-safe Docker prune plus targeted AI tooling hygiene: Claude VM + transcripts, Codex session transcripts and cache, Cursor Electron caches/logs (workspace chat state kept) — wrapped in a deep-clean toolkit that replaces CleanMyMac, AppCleaner, DaisyDisk and iStat Menus as a side effect.

  • Docker, without losing your DB

    Runs system, image and builder prune in one shot. Never touches volumes, so Postgres, Redis and friends stay intact. Typical reclaim: 10–50 GB.

  • AI agents that don’t hoard

    Claude Desktop: clears the sandbox VM (vm_bundles) plus Electron caches when it is safe; Claude Code: removes JSONL transcripts under ~/.claude/projects, generic caches/logs, plugin caches and (where used) claude-mem’s Chroma DB. Codex CLI: rotates ~/.codex/archived_sessions, active sessions/, logs, shell snapshots, tmp and vendor import caches — keeps config.toml, memories, rules, skills, plugins and sqlite/ auth intact. Cursor: trims HTTP/GPU/code caches and logs (including ~/.cursor/ai-tracking); skips User/workspaceStorage and User/History so your IDE chats and local file timeline stay put.

And the deep-clean toolkit that comes with it

  • All-in-one toolkit

    Clean, uninstall, optimize, analyze, status — all from one binary.

  • Deep cleaning

    Caches, logs, browser leftovers, orphaned app data. Whitelist-aware.

  • Smart uninstaller

    Removes apps with launch agents, prefs, and hidden remnants.

  • Disk insights

    Interactive TUI that visualizes usage and pinpoints large files.

  • Live monitoring

    Real-time CPU, GPU, memory, disk, and network in the terminal.

Where the gigabytes hide

The paths your tools never tell you about.

Sandbox disk images, vector DBs, Claude & Codex session payloads, Electron caches under Cursor’s Application Support folder. Anything irreplaceable is skipped — Cursor workspace chat folders, Claude/Codex auth + memories where applicable, Docker volumes.

Path What it holds Typical size
~/Library/Application Support/Claude/vm_bundles Claude Desktop sandbox VM (rootfs + sessiondata) ~10 GB
~/.claude/projects Claude Code session transcripts (JSONL) 500 MB+
~/.codex/archived_sessions Codex CLI archived session transcripts 600 MB+
~/.codex/sessions Codex CLI active session transcripts varies
~/.cache/chroma claude-mem vector database ~150 MB
~/Library/Application Support/Cursor/
(Cache, CachedData, Code Cache, GPUCache, Dawn*, logs, User/logs)		 Cursor rebuildable Electron caches & log streams (chat workspace dirs excluded) tens of MB
~/.cursor/ai-tracking Cursor CLI-side AI telemetry scratch files < few MB
docker system / image / builder prune Stopped containers, unused images, build cache 10–50 GB

Preserved: Docker volumes · Claude ~/.claude/memory, ~/.claude/plugins/data, ~/.claude/skills · Cursor extensions/, User/workspaceStorage, User/History, editor settings/snippet files (User/settings.json, keybindings, etc.) · Codex config.toml, memories/, rules/, skills/, plugin trees and sqlite/ auth.

Get started

Install in one line.

The installer fetches signed binaries from the latest GitHub Release, verifies SHA-256 checksums, and drops mo/mole in /usr/local/bin.

Install macOS · arm64 & amd64 
curl -fsSL https://raw.githubusercontent.com/ricciviero/Mole/main/install.sh | bash
First run always preview first 
mo clean --dry-run    # preview only
mo analyze            # visual disk explorer
mo status             # live system dashboard
Update re-run the installer 
curl -fsSL https://raw.githubusercontent.com/ricciviero/Mole/main/install.sh | bash
Uninstall clean removal 
mo remove
